# Meeting Assistant

Persian (Farsi) AI meeting assistant — record, transcribe, diarize, and summarize meetings into structured artifacts (action items, decisions, minutes, Q&A, follow-up email draft, and more). Single-user, self-hosted, local SQLite.

> **Stack:** FastAPI · SQLAlchemy 2.0 async · Next.js 16 (App Router) · Tailwind 4 · shadcn/ui · ElevenLabs Scribe v2 · OpenRouter (Gemini 3 Flash Preview)

---

## Features

### Three ingest modes
1. **Upload** an existing recording (mp3 / wav / m4a / webm …).
2. **Record from microphone** in the browser.
3. **Capture another tab** (Google Meet, Zoom Web, Teams Web) via `getDisplayMedia` — with optional mic mix.

Live modes run **simultaneously** with:
- Realtime captions via ElevenLabs Scribe v2 Realtime (WebSocket).
- Batch diarization via Scribe v2 on stop (the authoritative transcript — realtime has no speaker diarization).

### Seven summary artifacts
Generated by Gemini 3 Flash Preview via OpenRouter with strict `json_schema` enforcement:

- Executive summary
- Action items (assignee + due date)
- Decisions
- Speaker-attributed minutes (built server-side from diarized words — not LLM-generated)
- Q&A
- Open questions / parking lot
- Follow-up email draft (formal or casual tone)

### Recurring meeting **Series**
Bind related meetings together for shared:
- **Glossary** — fed to Scribe as keyterms for better recognition of names, acronyms, and jargon (1000 × 50chars batch, 50 × 20chars realtime).
- **Speaker-name memory** — display names auto-suggested on the next meeting.
- **Email tone preference** — formal or casual.

Fuzzy series suggester (`rapidfuzz`) auto-recommends matching series on upload.

### Cross-cutting **Tags**
Flat M:N labels, independent of series. Filter the home page by tag, series, or free-text search.

### Per-meeting **Chat**
SSE-streamed chat against a finished meeting's full context (diarized transcript + every summary artifact + speaker names).

## Quick start

### Linux server (Ubuntu, pm2) — recommended for self-hosting

```bash
git clone https://github.com/sepehr071/meeting-assistant.git
cd meeting-assistant

# 1. one-time prereqs (apt + uv + node 22 + pnpm + pm2)
chmod +x bootstrap.sh && ./bootstrap.sh

# 2. fill API keys
cp backend/.env.example backend/.env
nano backend/.env          # ELEVENLABS_API_KEY, OPENROUTER_API_KEY, SESSION_SECRET (>=32 chars)

cp frontend/.env.example frontend/.env.local

# 3. boot it (installs deps, runs migrations, builds, starts pm2)
chmod +x setup.sh && ./setup.sh

# Custom ports / IP:
BACK_PORT=8001 FRONT_PORT=3001 ./setup.sh
SERVER_IP=192.168.1.50 ./setup.sh
```

Full deployment guide in [`INSTALL.md`](./INSTALL.md).

### Local development (Windows / macOS / Linux)

```bash
# backend
cd backend
cp .env.example .env             # fill API keys
uv sync
uv run alembic upgrade head
uv run uvicorn app.main:app --host 0.0.0.0 --port 8000

# frontend (separate shell)
cd frontend
cp .env.example .env.local
pnpm install
pnpm dev                         # http://localhost:3000
```

Required:
- Python 3.12 + [`uv`](https://docs.astral.sh/uv/)
- Node 22 + `pnpm`
- `ffmpeg` (used for audio duration probing)

## How it works

```
┌──────────┐    audio    ┌──────────────────┐   words+spk   ┌────────────┐    JSON    ┌──────────┐
│ Upload / │ ─────────▶ │ Scribe v2 batch  │ ────────────▶ │ Gemini 3   │ ─────────▶ │ Summary  │
│ Record   │            │  (diarize=True)  │               │ Flash      │            │ + Minutes│
└──────────┘            └──────────────────┘               │ via        │            └──────────┘
     │                                                     │ OpenRouter │
     │   (live mode only)                                  └────────────┘
     │   ┌──────────────────────┐
     └──▶│ Scribe v2 Realtime    │ ──── live captions (display-only, discarded on stop)
         │ (WebSocket, PCM 16k)  │
         └──────────────────────┘
```

**State machine:** `uploaded → transcribing → summarizing → done` (or `failed`). Pipeline runs as a FastAPI `BackgroundTask`; idempotent — re-running on a `done` meeting is a no-op. In-flight processing can be cancelled mid-run.

## Configuration

### Backend (`backend/.env`)

| Var | Purpose |
|---|---|
| `ELEVENLABS_API_KEY` | Scribe v2 transcription |
| `OPENROUTER_API_KEY` | LLM access |
| `OPENROUTER_MODEL` | Default `google/gemini-3-flash-preview` |
| `SESSION_SECRET` | Cookie signing key (≥32 chars) |
| `ALLOWED_ORIGIN` | Comma-separated CORS origins |
| `ALLOWED_ORIGIN_REGEX` | Regex override for CORS |
| `SESSION_SAME_SITE` | `lax` for same-site cross-port, `none` for cross-host (needs HTTPS) |
| `SESSION_COOKIE_SECURE` | `true` for HTTPS deployments |
| `STORAGE_DIR` | Audio storage path (default `./storage`) |
| `DATABASE_URL` | SQLite path (default `sqlite+aiosqlite:///./meeting.db`) |
| `HOST` / `PORT` | Server bind |

### Frontend (`frontend/.env.local`)

| Var | Purpose |
|---|---|
| `NEXT_PUBLIC_API_BASE` | Backend URL incl. `/api` suffix (baked at build time) |
| `PORT` | Next.js listen port |

## Constraints (intentional non-goals)

- **Single user, no auth.** No multi-tenancy.
- **Persian only** by default (`language_code=fas`). Flip in `app/services/transcription.py` for another single language.
- **No live diarization** — speaker labels come from the post-stop batch pass.
- **Single-blob uploads** (capped 500MB). No chunked progressive upload.
- **No background workers** (Redis/Celery). FastAPI `BackgroundTasks` is enough for single-user load.
- **Audio retained indefinitely** under `backend/storage/audio/`. No cleanup job.

---

## Browser support

- **Chrome / Edge:** full support (mic + tab capture + live captions).
- **Firefox:** mic + upload work; tab-audio support is partial.
- **Safari:** untested.

Mic + tab capture require a **secure context** — HTTPS or `localhost`. On plain HTTP over a LAN IP the browser blocks them; uploads still work.
